# 计算机网络 — 知识点详解

---

## 1. 计算机网络和网络体系结构的概念

### 1.1 计算机网络的基本概念

- **计算机网络**：将地理位置不同的、具有独立功能的多台计算机及其外部设备，通过通信线路和通信设备连接起来，在网络操作系统和通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。
- **网络的功能**：数据通信、资源共享、分布式处理、提高可靠性。

### 1.2 网络的分类

- **按覆盖范围**：
  - LAN（局域网）：几十米到几公里。
  - MAN（城域网）：一个城市范围。
  - WAN（广域网）：跨越城市、国家。
  - PAN（个人区域网）：10米以内。
- **按拓扑结构**：星形、总线形、环形、网状。
- **按使用者**：公用网、专用网。

### 1.3 网络体系结构

- **协议（Protocol）**：为网络中的数据交换而建立的规则、标准或约定。
  - 三要素：**语法**（数据格式）、**语义**（控制信息）、**时序**（事件顺序）。
- **分层**：将复杂的网络功能分解为若干层，每层实现特定功能。
- **服务**：下层为上层提供的功能。
  - **面向连接的服务**：建立连接→数据传输→释放连接（可靠）。
  - **无连接的服务**：直接发送数据（不可靠但灵活）。
- **服务访问点（SAP）**：上层访问下层服务的接口。

### 1.4 OSI 七层模型

| 层号 | 名称 | 功能 | 数据单元 |
|------|------|------|----------|
| 7 | 应用层 | 为用户应用提供网络服务 | 报文 |
| 6 | 表示层 | 数据格式转换、加密、压缩 | 报文 |
| 5 | 会话层 | 建立、管理和终止会话 | 报文 |
| 4 | 传输层 | 端到端可靠传输、流量控制 | 报文段 |
| 3 | 网络层 | 路由选择、拥塞控制 | 分组/包 |
| 2 | 数据链路层 | 成帧、差错控制、流量控制 | 帧 |
| 1 | 物理层 | 传输比特流 | 比特 |

### 1.5 TCP/IP 四层模型

| TCP/IP 层 | 对应 OSI 层 | 典型协议 |
|-----------|------------|----------|
| 应用层 | 应用层+表示层+会话层 | HTTP, FTP, SMTP, DNS, Telnet |
| 传输层 | 传输层 | TCP, UDP |
| 网际层 | 网络层 | IP, ICMP, ARP, IGMP |
| 网络接口层 | 数据链路层+物理层 | Ethernet, PPP, Wi-Fi |

## 2. 数据通信的基础知识

### 2.1 基本概念

- **数据**：传递信息的实体。
- **信号**：数据的物理表示。
  - **模拟信号**：连续变化。
  - **数字信号**：离散取值。
- **信道**：信号传输的通道。
- **带宽**：
  - 模拟通信：信号频带宽度（Hz）。
  - 数字通信：最大数据传输速率（bps）。
- **码元**：数字信号的一个状态符号。
- **波特率（Baud Rate）**：每秒传输的码元数。
- **比特率（Bit Rate）**：每秒传输的比特数。比特率 = 波特率 × log₂V（V为码元状态数）。

### 2.2 奈奎斯特定理与香农定理

- **奈奎斯特定理（理想无噪声信道）**：
  - 最大码元传输率 = 2W（Baud），W 为信道带宽（Hz）。
  - 最大数据传输率 = 2W × log₂V（bps）。

- **香农定理（有噪声信道）**：
  - 信道容量 C = W × log₂(1 + S/N)（bps）
  - S/N 为信噪比，通常用 dB 表示：信噪比(dB) = 10 × log₁₀(S/N)。

### 2.3 编码与调制

**数字数据 → 数字信号（编码）**：
- **非归零编码（NRZ）**：高电平=1，低电平=0。
- **曼彻斯特编码**：每个位中间跳变，从低到高=1，从高到低=0（或反之）。自同步。
- **差分曼彻斯特编码**：每位中间跳变提供同步，位开始有跳变=0，无跳变=1。

**数字数据 → 模拟信号（调制）**：
- **调幅（ASK）**：改变载波振幅。
- **调频（FSK）**：改变载波频率。
- **调相（PSK）**：改变载波相位。
- **正交振幅调制（QAM）**：同时改变振幅和相位。

### 2.4 信道复用技术

- **频分复用（FDM）**：不同用户占用不同频带。
- **时分复用（TDM）**：不同用户占用不同时隙。
- **统计时分复用（STDM）**：按需动态分配时隙。
- **波分复用（WDM）**：光纤中的频分复用（不同波长）。
- **码分复用（CDMA）**：不同用户使用不同的编码（正交码片序列）。

### 2.5 数据交换技术

- **电路交换**：通信前建立专用通路，通信结束后释放。适合大量连续数据传输。
- **报文交换**：以整个报文为单位存储转发。
- **分组交换**：
  - **数据报**：每个分组独立路由，可能走不同路径（如 IP）。
  - **虚电路**：先建立虚电路，分组沿虚电路传输（如 ATM）。

### 2.6 差错控制

- **检错码**：
  - **奇偶校验**：加 1 位使 1 的个数为奇/偶。只能检测奇数个错误。
  - **CRC（循环冗余校验）**：用生成多项式除数据得余数作为校验码。检测能力强。
- **纠错码**：
  - **海明码**：能纠正 1 位错误。需要 r 个校验位满足 2^r ≥ m+r+1（m 为数据位）。

## 3. Internet 的体系结构和各层协议

### 3.1 物理层

- 传输介质：双绞线、同轴电缆、光纤、无线介质。
- 物理层设备：中继器（Repeater）、集线器（Hub）。

### 3.2 数据链路层

**功能**：成帧、差错检测、流量控制、介质访问控制。

**成帧方法**：字符计数法、字符填充首尾定界符法、比特填充首尾定界符法。

**以太网（IEEE 802.3）**：
- **CSMA/CD 协议**（载波监听多路访问/冲突检测）：
  1. 先听后发：发送前先监听信道是否空闲。
  2. 边发边听：发送过程中持续检测冲突。
  3. 冲突停止：检测到冲突后立即停止发送。
  4. 随机重发：等待随机时间后重试（二进制指数退避算法）。
- **最小帧长**：64 字节（确保在发送完之前能检测到冲突）。
- **MAC 地址**：48 位全球唯一地址。
- 以太网交换机：根据 MAC 地址转发帧，全双工，无冲突。

**其他协议**：
- **PPP（点对点协议）**：面向字符的协议，支持多种网络层协议，有认证功能（PAP/CHAP）。
- **生成树协议（STP）**：防止交换网络中的环路。（2025 真题）
- **VLAN（虚拟局域网）**：逻辑上划分的广播域。

### 3.3 网络层（IP 层）

**IP 协议（IPv4）**：
- **IP 地址**：32 位，网络号 + 主机号。
  - A 类：1.0.0.0 ~ 126.255.255.255（/8），网络号 8 位。
  - B 类：128.0.0.0 ~ 191.255.255.255（/16），网络号 16 位。
  - C 类：192.0.0.0 ~ 223.255.255.255（/24），网络号 24 位。
  - D 类（组播）：224.0.0.0 ~ 239.255.255.255
  - E 类（保留）：240.0.0.0 ~ 255.255.255.255
- **子网划分**：从主机号借位作为子网号。子网掩码标识网络位和主机位。
- **CIDR（无类域间路由）**：取消传统的 ABC 分类，用前缀长度表示网络号。如 192.168.1.0/24。
  - **路由聚合（超网）**：将多个连续网络合并为一个更大的网络。
- **特殊 IP 地址**：
  - 0.0.0.0：本机。
  - 127.x.x.x：环回地址。
  - 255.255.255.255：广播地址。
  - 10.x.x.x, 172.16.x.x~172.31.x.x, 192.168.x.x：私有地址。

**IP 数据报格式**：
- 首部长度（20~60字节）、总长度、TTL（生存时间）、协议号（TCP=6, UDP=17, ICMP=1）等。

**NAT（网络地址转换）**：将私有 IP 转换为公有 IP，解决 IP 地址不足问题。

**ICMP（互联网控制报文协议）**：
- 用于报告差错和传递控制信息。
- 类型：目的不可达、超时、重定向、Echo 请求/应答（ping）。
- **traceroute** 利用 ICMP 超时报文探测路由路径。

**ARP（地址解析协议）**：将 IP 地址解析为 MAC 地址。
- RARP：反向，MAC → IP（已很少使用）。

**路由选择算法与协议**：

- **静态路由**：手工配置。
- **动态路由**：
  - **距离向量路由（RIP）**：
    - 基于 Bellman-Ford 算法。
    - 以跳数为度量，最大 15 跳。
    - 每 30 秒向邻居广播路由表。
    - 问题：慢收敛、计数到无穷。
    - 解决：水平分裂、毒性逆转、触发更新。
  - **链路状态路由（OSPF）**：
    - 基于 Dijkstra 算法。
    - 每个路由器维护完整的网络拓扑图。
    - 通过泛洪（Flooding）传播链路状态通告（LSA）。
    - 支持区域划分、支持 VLSM。
  - **路径向量路由（BGP）**：
    - 外部网关协议（EGP），用于自治系统间路由。
    - 基于策略选择路径，考虑经济、政治因素。

**IPv6**：
- 128 位地址，简化首部，取消校验和。
- 地址表示：冒号十六进制，如 2001:0db8::1。

> **真题考点：IP 地址与路由**
> - CIDR 192.168.1.0/22 表示前 22 位为网络号，覆盖 192.168.0.0 ~ 192.168.3.255。（2025 真题）
>   - 192.168.2.2 与 192.168.3.3 属于同一网段（B）；192.168.1.1 与 192.168.4.1 不属于（跨越了 /22 边界）。
> - OSPF 使用**链路状态信息**计算最短路径（C），基于 Dijkstra 算法，属于内部网关协议。（2025 真题）

> **真题考点：CSMA/CA**
> - CSMA/CA 全称：载波监听多路访问/**冲突避免**（Collision Avoidance），用于无线局域网（802.11），通过 ACK 确认数据帧正确接收。（2025 真题）

> **真题考点：DNS 域名解析**
> - 递归解析：客户端向本地域名服务器发送查询请求，服务器负责完成全部查询过程后返回结果。（2025 真题）

> **真题考点：防火墙**
> - 防火墙基于 **IP 地址和端口号**等规则过滤数据包（B），工作在网络层/传输层，不能防止所有攻击。（2025 真题）

### 3.4 传输层

**端口**：标识主机上的应用进程，16 位（0~65535）。
- 知名端口：0~1023（如 HTTP=80, FTP=21, SMTP=25, DNS=53）。

**TCP（传输控制协议）**：
- **面向连接、可靠传输、全双工、字节流**。
- **TCP 报文段格式**：源端口、目的端口、序号、确认号、数据偏移、标志位（SYN, ACK, FIN, RST）、窗口大小、校验和等。

**三次握手（建立连接）**：
1. 客户端 → 服务器：SYN=1, seq=x
2. 服务器 → 客户端：SYN=1, ACK=1, seq=y, ack=x+1
3. 客户端 → 服务器：ACK=1, seq=x+1, ack=y+1

**四次挥手（释放连接）**：
1. A → B：FIN=1, seq=u
2. B → A：ACK=1, ack=u+1
3. B → A：FIN=1, seq=w
4. A → B：ACK=1, ack=w+1

**TCP 可靠传输机制**：
- **序号和确认**：每个字节有序号，接收方确认已正确收到的数据。
- **超时重传**：RTO（Retransmission Timeout），动态计算。
- **滑动窗口**：实现流量控制，窗口大小动态调整。
- **拥塞控制**：
  - **慢启动**：拥塞窗口从 1 MSS 开始指数增长。
  - **拥塞避免**：达到慢启动阈值后线性增长。
  - **快重传**：收到 3 个重复 ACK 立即重传（不等超时）。
  - **快恢复**：将慢启动阈值减半，拥塞窗口设为新阈值。

**UDP（用户数据报协议）**：
- **无连接、不可靠、面向报文、开销小**。
- 首部仅 8 字节（源端口、目的端口、长度、校验和）。
- 适合实时应用（视频、语音）和 DNS 查询等。

> **真题考点：传输层**
> - A 发送 seq=300, ack=200, 有效载荷 100 字节 → B 返回 **ack=400, seq=200**（确认号 = 对方 seq+载荷长度）。（2025 真题）
> - UDP 端口号的作用是**区分不同应用程序**（B）。（2025 真题）
> - TCP 拥塞控制：收到 3 个重复 ACK → 快重传，慢启动阈值减半，拥塞窗口设为新阈值（快恢复）。（2025 真题）

> **真题考点：信道利用率计算**
> - 停止-等待协议利用率 U = (帧长/传输速率) / (帧长/传输速率 + 2×传播时延)
> - 例：120km 信道，RTT=20ms，10Mbps，利用率 = (帧长/10Mbps)/(帧长/10Mbps + 20ms)。（2025 真题）
> - 信道利用率与帧长、传输速率、传播时延（信道长度/信号速度）相关。

### 3.5 应用层

**DNS（域名系统）**：
- 将域名解析为 IP 地址。分布式、层次化数据库。
- 域名层次：根域 → 顶级域（.com, .cn, .edu）→ 二级域 → …
- 查询方式：递归查询、迭代查询。
- 资源记录：(域名, 类型, 值)，类型有 A（IPv4）、AAAA（IPv6）、CNAME（别名）、MX（邮件）、NS（域名服务器）。

**HTTP（超文本传输协议）**：
- 基于 TCP，默认端口 80。无状态协议。
- HTTP/1.0：非持久连接（每个对象一个 TCP 连接）。
- HTTP/1.1：持久连接（一个 TCP 连接传输多个对象），管道化。
- HTTP/2.0：多路复用、头部压缩、服务器推送。
- 请求方法：GET, POST, PUT, DELETE, HEAD 等。
- 状态码：200 OK, 301 永久重定向, 304 未修改, 404 未找到, 500 服务器错误。

**FTP（文件传输协议）**：
- 基于 TCP，使用两个连接：控制连接（端口 21）+ 数据连接（端口 20）。

**SMTP/POP3/IMAP（电子邮件）**：
- SMTP：发送邮件，端口 25。
- POP3：接收邮件，端口 110（下载并删除）。
- IMAP：接收邮件，端口 143（在线管理）。

**DHCP（动态主机配置协议）**：
- 基于 UDP，自动分配 IP 地址、子网掩码、默认网关、DNS 服务器。
- 过程：Discover → Offer → Request → ACK。

## 4. 网络互联技术

### 4.1 网络互联设备

| 设备 | 工作层次 | 功能 |
|------|---------|------|
| 中继器/集线器 | 物理层 | 信号放大和转发 |
| 网桥/交换机 | 数据链路层 | 基于 MAC 地址转发帧 |
| 路由器 | 网络层 | 基于 IP 地址路由转发 |
| 网关 | 应用层 | 协议转换 |

### 4.2 交换技术

- **二层交换**：基于 MAC 地址表转发帧。
- **三层交换**：结合二层交换速度和三层路由功能。
- **生成树协议（STP）**：防止交换网络中的环路。

### 4.3 VPN（虚拟专用网络）

- 在公共网络上建立安全的私有通道。
- 技术：IPSec、SSL/TLS VPN、MPLS VPN。

## 5. 典型网络应用

- **Web 服务**：HTTP 协议，B/S 架构。

- **流媒体**：RTP/RTCP、RTSP。

## 6. 网络安全的概念和协议

### 6.1 安全基本概念

- **安全目标（CIA 三元组）**：
  - **机密性（Confidentiality）**：防止信息泄露。
  - **完整性（Integrity）**：防止信息被篡改。
  - **可用性（Availability）**：保证服务可用。
- **安全威胁**：窃听、篡改、伪造、拒绝服务（DoS）、重放攻击、中间人攻击。

### 6.2 密码学基础

**对称加密**：加密和解密使用同一密钥。
- 算法：DES（56位密钥）、3DES、AES（128/192/256位密钥）。
- 优点：速度快。缺点：密钥分发困难。

**非对称加密**：公钥加密，私钥解密（或反过来）。
- 算法：RSA、ElGamal、ECC。
- 优点：无需共享密钥。缺点：速度慢。

**数字签名**：用发送方的私钥签名，接收方用公钥验证。保证真实性、不可否认性。

**数字证书与 PKI**：
- CA（证书颁发机构）签发数字证书，绑定公钥与身份。
- X.509 证书标准。

**散列函数**：MD5（128位）、SHA-1（160位）、SHA-256。
- 用于验证数据完整性。
- HMAC：基于散列的消息认证码。

### 6.3 安全协议

**SSL/TLS**：
- 位于传输层和应用层之间，为 HTTP 等应用协议提供安全通道。
- TLS 握手：协商加密算法、交换密钥、验证证书。
- HTTPS = HTTP + TLS，端口 443。

**IPSec**：
- 网络层安全协议，保护 IP 数据包。
- 两种模式：传输模式（只保护数据）和隧道模式（保护整个 IP 包）。
- AH（认证头）：提供完整性和认证。
- ESP（封装安全载荷）：提供加密+认证。
- IKE（密钥交换）：用于协商安全关联（SA）。

**防火墙**：
- **包过滤防火墙**：基于源/目的 IP、端口、协议过滤。
- **状态检测防火墙**：跟踪连接状态。
- **应用层网关（代理防火墙）**：在应用层检查数据。

**入侵检测系统（IDS）**：
