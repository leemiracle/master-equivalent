# 计算机系统结构 — 知识点详解

---

## 1. 计算机系统结构的基本概念

### 1.1 计算机系统结构的定义

- **计算机系统结构（Architecture）**：程序员看到的计算机系统的属性，即机器语言程序员（或编译程序设计者）所看到的计算机系统的概念性结构与功能特性。
- **计算机组成（Organization）**：计算机系统结构的逻辑实现，包括数据流、控制流的组成及逻辑设计。
- **计算机实现（Implementation）**：计算机组成的物理实现，包括器件、线路等。

### 1.2 计算机系统的设计方法

- **从上往下设计**：先确定软件系统，再设计硬件。
- **从下往上设计**：先设计硬件，再开发软件。
- **从中间开始设计**：先确定软硬件交界面（指令集），然后分别向上和向下设计。

### 1.3 计算机系统结构的分类

**Flynn 分类法**（按指令流和数据流分类）：

| 类型 | 指令流 | 数据流 | 典型代表 |
|------|--------|--------|----------|
| SISD | 单一 | 单一 | 传统单处理机 |
| SIMD | 单一 | 多个 | 向量处理机、阵列机 |
| MISD | 多个 | 单一 | 很少（理论模型） |
| MIMD | 多个 | 多个 | 多处理机、多计算机 |

### 1.4 计算机系统的性能评价

- **CPU 时间** = 指令数 × CPI × 时钟周期时间 = (指令数 × CPI) / 主频
- **CPI（Cycles Per Instruction）**：每条指令的平均时钟周期数。
- **MIPS** = 指令数 / (执行时间 × 10⁶) = 主频 / (CPI × 10⁶)
- **MFLOPS/GFLOPS/TFLOPS**：每秒百万/十亿/万亿次浮点运算。
- **Amdahl 定律**：
  - 加速比 S = 1 / ((1-f) + f/Se)
  - 其中 f 为可改进部分的比例，Se 为该部分的加速倍数。
  - **结论**：加速比的上限为 1/(1-f)，即使某部分无限加速，整体加速也受限于不可改进部分。
- **吞吐量**：单位时间内完成的任务数。
- **基准测试程序**：SPEC、Linpack 等。

### 1.5 计算机系统结构的发展历程

- 电子管 → 晶体管 → 集成电路 → 大规模集成电路 → 超大规模集成电路
- **摩尔定律**：集成电路上的晶体管数目大约每 18-24 个月翻一番。

## 2. 数据表示与指令系统

### 2.1 数据表示

- **数据表示**：硬件能直接识别和引用的数据类型。
- **数据类型** vs **数据表示**：数据类型是软件层面的概念，数据表示是硬件层面支持的类型。

**基本数据表示**：
- **定点数**：
  - 原码：最高位为符号位，其余为绝对值。
  - 补码：正数同原码，负数为反码+1。最常用的表示法。
  - 反码：正数同原码，负数为各位取反。
  - 移码：补码符号位取反，用于浮点数的阶码表示。
- **浮点数**：N = M × R^E
  - M（尾数）、R（基数，通常为2）、E（阶码）。
  - IEEE 754 标准：
    - 单精度（32位）：1位符号 + 8位阶码 + 23位尾数
    - 双精度（64位）：1位符号 + 11位阶码 + 52位尾数
    - 阶码用移码表示，偏移量分别为 127 和 1023。
    - 规格化：尾数最高有效位为 1（隐含表示，省略 1 位）。
    - 特殊值：全0阶码表示非规格化数/零，全1阶码表示无穷大/NaN。

**自定义数据表示**：
- **带标志符的数据表示**：数据自带类型标志，由硬件解释。
- **数据描述符**：用于描述复杂数据结构（如数组）。

### 2.2 寻址方式

- **立即寻址**：操作数直接在指令中。
- **直接寻址**：指令中给出操作数的有效地址。
- **间接寻址**：指令中给出操作数地址的地址。
- **寄存器寻址**：操作数在寄存器中。
- **寄存器间接寻址**：寄存器中存放操作数地址。
- **偏移寻址**：有效地址 = 寄存器内容 + 偏移量。
  - **相对寻址**：相对于 PC 的偏移。
  - **基址寻址**：基址寄存器 + 偏移量。
  - **变址寻址**：变址寄存器 + 偏移量。
- **比例变址寻址**：有效地址 = 基址 + 变址 × 比例因子 + 偏移量。

### 2.3 指令系统设计

**指令格式**：操作码 + 地址码
- 操作码扩展技术：可变长操作码（短操作码用于常用指令）。

**指令系统设计原则**：
- 完备性、有效性、规整性、兼容性。

**指令的分类**：
- 数据传送指令
- 算术运算指令
- 逻辑运算指令
- 移位指令
- 控制转移指令
- 输入输出指令

## 3. CISC 与 RISC

### 3.1 CISC（复杂指令集计算机）

- 指令数量多、格式多样、长度可变。
- 寻址方式多样。
- 指令执行时间差异大。
- 微程序控制为主。
- 代表：x86 架构（Intel、AMD）。
- 问题：指令使用频率不均（80/20规则：20%的指令占80%的使用）。

### 3.2 RISC（精简指令集计算机）

**设计原则**：
1. 只用频率高的指令。
2. 指令长度固定，格式规整。
3. 只有 Load/Store 指令访问存储器，其余指令在寄存器间操作。
4. 寻址方式少。
5. 硬布线控制为主。
6. 优化编译器。

**RISC 的关键技术**：
- **寄存器窗口**：用于过程调用时参数传递，减少访存。
- **延迟转移（分支延迟槽）**：在转移指令后安排一条总是执行的指令。
- **指令流水线**：RISC 更适合流水线实现。
- **优化编译器**：编译器负责指令调度、寄存器分配。

代表：MIPS、ARM、RISC-V、SPARC。

### 3.3 CISC 与 RISC 的对比

| 特征 | CISC | RISC |
|------|------|------|
| 指令数量 | 多（>200） | 少（<100） |
| 指令长度 | 可变 | 固定 |
| 寻址方式 | 多 | 少 |
| CPI | 大（>2） | 小（≈1） |
| 控制方式 | 微程序 | 硬布线 |
| 寄存器 | 少 | 多 |

## 4. 存储系统

### 4.1 层次存储系统的基本原理

**存储体系**：寄存器 → Cache（L1/L2/L3）→ 主存 → 辅存（磁盘）→ 后援存储（磁带）
- 上一层容量小、速度快、价格高。
- 下一层容量大、速度慢、价格低。

**程序局部性原理**：
- **时间局部性**：最近访问的数据在近期还可能被访问。
- **空间局部性**：访问某地址后，其附近地址也可能被访问。

### 4.2 并行存储器

- **多体交叉存储器**：将存储器分成多个模块（bank），可以并行或交叉访问。
  - **高位交叉**：高位地址选体。同一模块地址连续，利于扩展容量。
  - **低位交叉**：低位地址选体。连续地址分布在不同模块，利于流水线式并行访问。

### 4.3 高速缓冲存储器（Cache）

**基本原理**：利用局部性，在 CPU 和主存之间设置小容量高速存储器。

**地址映像与变换**：
- **直接映像**：主存块只能映像到 Cache 的一个特定行。
  - 主存地址 = 标记（Tag）+ 行号（Index）+ 块内地址
  - 优点：实现简单。缺点：冲突率高。
- **全相联映像**：主存块可以映像到 Cache 的任意行。
  - 优点：冲突率低。缺点：查找慢，成本高。
- **组相联映像**：先按组直接映像，组内全相联。
  - n 路组相联：每组 n 行。是前两者的折中。

**替换算法**：
- **RAND（随机）**：随机替换。
- **FIFO（先进先出）**：替换最早装入的块。可能产生 Belady 异常。
- **LRU（最近最少使用）**：替换最久未被访问的块。最优的实用算法。
- **OPT（最优替换）**：替换将来最长时间不用的块。理论最优，不可实现。

**Cache 的性能**：
- **命中率 h** = 命中次数 / 总访问次数
- **平均访问时间** = h × t_c + (1-h) × t_m
- **加速比** = t_m / (h × t_c + (1-h) × t_m)

> **真题考点：Cache 计算（高频）**
> - Cache 命中率 95%，Cache 访问时间 10ns，主存访问时间 100ns → 平均访存时间 = 0.95×10 + 0.05×100 = 14.5ns
> - 若要求平均访存时间 ≤ 20ns，命中率至少需 h：h×10 + (1-h)×100 ≤ 20 → h ≥ 8/9 ≈ 88.9%
> - **组相联映射计算**：主存 256KB，Cache 8KB，二路组相联，块 32B → Cache 行数 = 8KB/32B = 256，组数 = 256/2 = 128，组号字段 = log₂128 = 7 位，块内地址 = log₂32 = 5 位。（2025 真题）

**写策略**：
- **写直达（Write Through）**：同时写 Cache 和主存。
- **写回法（Write Back）**：只写 Cache，替换时才写回主存（需脏位）。

### 4.4 虚拟存储器

**基本概念**：将主存和辅存统一编址，给用户提供比实际主存大得多的地址空间。

- **虚拟地址（逻辑地址）**：程序使用的地址。
- **物理地址（实地址）**：实际主存地址。
- **地址变换**：MMU（存储管理部件）将虚拟地址转换为物理地址。

**页式虚拟存储器**：
- 将虚拟空间和物理空间都划分为固定大小的**页**。
- **页表**：记录虚拟页号到物理页号的映射。
- **TLB（Translation Lookaside Buffer）**：页表的 Cache，加速地址变换。
- **缺页处理**：访问的页不在主存时，产生缺页异常，从辅存调入。

**段式虚拟存储器**：按程序逻辑分段，段长度可变。

**段页式虚拟存储器**：先分段，段内再分页。结合段式和页式优点。

### 4.5 网络存储

- **DAS（直连式存储）**：直接连接到服务器。
- **NAS（网络附加存储）**：通过网络提供文件级存储服务。
- **SAN（存储区域网络）**：高速专用网络提供块级存储服务。

## 5. 输入输出系统

### 5.1 基本输入输出方式

- **程序查询方式**：CPU 轮询 I/O 设备状态，效率最低。
- **程序中断方式**：I/O 设备准备好后向 CPU 发中断请求，CPU 响应中断进行数据传输。
- **DMA 方式**：DMA 控制器直接管理数据在 I/O 设备和主存之间的传输，不经过 CPU。
- **通道方式**：通道处理机独立执行通道程序，管理 I/O 操作。

### 5.2 中断系统

- **中断分类**：
  - 内中断（异常）：CPU 内部产生（除法溢出、缺页、断点等）。
  - 外中断（中断）：外部设备产生。
- **中断处理过程**：中断请求 → 中断响应 → 保存现场 → 中断服务 → 恢复现场 → 中断返回。

**软硬件功能分配**：
- **中断响应**：硬件完成（发现中断、确定优先级、保存断点）。
- **中断处理**：软件完成（中断服务程序）。

**中断优先级**：
- 多个中断同时发生时，优先响应高优先级中断。
- 可以设置**中断屏蔽字**来改变优先级和允许嵌套。

### 5.3 通道处理机

- **字节多路通道**：以字节为单位交叉服务多台慢速设备。
- **数组选择通道**：以块为单位为一台高速设备服务，一段时间只服务一台。
- **数组多路通道**：结合前两者，以块为单位交叉服务多台高速设备。

### 5.4 基本存储设备

- **磁表面存储器**：磁盘（HDD）。
- **固态存储器**：SSD（闪存）。
- 关键参数：容量、寻道时间、旋转延迟、数据传输率。

## 6. 流水线处理机

### 6.1 流水线的工作原理

- **基本思想**：将一条指令的执行过程分成多个阶段，不同指令的不同阶段可以重叠执行。
- **典型五级流水线**：取指(IF) → 译码(ID) → 执行(EX) → 访存(MEM) → 写回(WB)。

**流水线性能指标**：
- **吞吐率 TP** = 完成的任务数 / 总时间 ≈ 1/Δt（达到稳定状态后）
- **加速比 S** = 非流水线时间 / 流水线时间 ≈ k（流水段数）
- **效率 E** = 有效时空区面积 / 总时空区面积

> **真题考点：流水线性能计算（高频）**
> - 流水线吞吐率 = n / (ΣΔtᵢ + (n-1)×max(Δtᵢ))，其中各段时间不等时，取最慢段为瓶颈。
> - 例：5 段流水线各段时间 1ns, 2ns, 1ns, 3ns, 1ns，连续 100 条指令：
>   - 瓶颈段 = 3ns，总时间 = (1+2+1+3+1) + 99×3 = 8 + 297 = 305ns
>   - 实际吞吐率 = 100/305 ≈ 0.33 条/ns（2025 真题）

> **真题考点：性能计算**
> - CPU 执行时间 = 指令数 × CPI / 主频
> - 例：10⁶ 条指令，50% 算术（2 周期）、30% 访存（4 周期）、20% 控制（3 周期），CPI = 0.5×2+0.3×4+0.2×3 = 2.8，2GHz 下执行时间 = 10⁶×2.8/(2×10⁹) = 1.4ms，MIPS = 10⁶/1.4ms ≈ 714。（2025 真题）

### 6.2 流水线的分类

- **单功能/多功能流水线**
- **静态/动态流水线**：静态同一时间只能一种功能，动态可以多种。
- **线性/非线性流水线**
- **标量/向量流水线**

### 6.3 流水线相关性（冲突）

**结构相关（资源冲突）**：不同指令在同一时刻争用同一硬件资源。
- 解决：资源重复配置、指令停顿。

**数据相关**：
- **RAW（写后读）**：最常见，j 需要 i 写的结果。
- **WAR（读后写）**：j 写回 i 要读的位置。
- **WAW（写后写）**：j 和 i 写同一位置。
- 解决方法：
  - **数据转发（Forwarding/Bypassing）**：将结果直接从执行阶段传给需要的指令。
  - **编译器调度**：重新排列指令顺序，插入无关指令。
  - **流水线停顿（Stall/Bubble）**：插入空操作。

**控制相关（分支冲突）**：分支指令改变指令流方向。
- 解决方法：
  - **分支预测**：静态预测（总是不跳/总是跳）和动态预测（BHT分支历史表、两位预测器）。
  - **延迟分支**：将分支指令后的延迟槽填入有用指令。
  - **预取两个方向**。

### 6.4 超标量处理机

- **单发射**：每个时钟周期发射一条指令。
- **多发射**：每个时钟周期发射多条指令。
  - **超标量**：硬件动态调度，多条指令并行执行。需要多条流水线。
  - **超流水线**：将每个流水段再细分，提高时钟频率。
  - **超标量超流水线**：两者结合。

**指令调度**：
- **静态调度**：编译器完成。
- **动态调度**：硬件完成（记分牌 Scoreboarding、Tomasulo 算法）。
  - **Tomasulo 算法**：使用保留站、寄存器换名、公共数据总线（CDB）解决 WAW 和 WAR 相关。

**乱序执行与按序完成**：
- 允许指令乱序执行以提高并行度，但需要按序完成以保证正确性。

## 7. 向量处理机

### 7.1 向量处理方式

- **横向处理**：逐个元素处理完一个向量操作再处理下一个。
- **纵向处理**：对所有元素执行同一操作。
- **纵横处理**：分组处理。

### 7.2 向量处理机结构

- **寄存器-寄存器型**：向量运算在向量寄存器间进行（如 CRAY-1）。
- **存储器-存储器型**：向量运算直接在存储器间进行。

### 7.3 提高向量处理机性能的方法

- **链接技术（Chaining）**：将一个向量操作的结果直接作为下一个向量操作的输入，形成流水线链。
- **向量循环（Strip Mining）**：当向量长度超过向量寄存器长度时，分段处理。
- **向量递归**：向量元素的递归计算。
- **条件执行语句**：向量掩码（掩码位控制哪些元素参与运算）。

### 7.4 向量处理机的性能评价

- **向量长度 n 时的执行时间** ≈ T_startup + n × T_result（流水线填满后每个周期出一个结果）
- **半性能长度 n₁/₂**：性能达到峰值一半所需的向量长度。
- **最大性能 R_∞**：向量长度趋于无穷时的渐近性能。

## 8. 互连网络

### 8.1 基本概念

- **互连网络**：连接多处理机/多计算机中各节点的网络。
- **作用**：实现处理器之间、处理器与存储器之间的数据传输。

### 8.2 互连函数

- **互连函数 f(x)**：描述输入端 x 连接到输出端 f(x) 的映射。
- 常见互连函数：
  - **恒等**：f(x) = x
  - **交换（Exchange）**：f(x) = x ⊕ 2^k（翻转第 k 位）
  - **全混洗（Perfect Shuffle）**：f(x) = (x << 1) mod (N-1)（循环左移一位）
  - **蝶式（Butterfly）**：交换最高位和最低位
  - **位反序（Bit Reversal）**：将二进制位反转

### 8.3 静态互连网络

- **线性阵列**：一维链式连接。
- **环**：首尾相连。
- **网格（Mesh）**：二维/三维阵列。
- **树形**：二叉树、胖树。
- **超立方体（Hypercube）**：n 维超立方体有 2^n 个节点，每个节点连 n 个邻居。
- **全连接**：每对节点之间都有直接连接。

### 8.4 动态互连网络

- **总线**：共享通信介质，简单但有带宽瓶颈。
- **交叉开关（Crossbar）**：N×N 交叉矩阵，无冲突但成本 O(N²)。
- **多级互连网络（MIN）**：
  - **Omega 网络**：使用 2×2 交换开关，全混洗+交换。
  - **Benes 网络**：可重排非阻塞网络。
  - **蝶形网络（Butterfly）**：多级交换网络。

### 8.5 消息传递机制

- **存储转发（Store-and-Forward）**：每个中间节点完整接收消息再转发。延迟与距离成正比。
- **虫孔路由（Wormhole Routing）**：消息分成小的 flit，流水线式传输。延迟与距离无关（基本无关）。
- **虚拟通道**：一条物理链路复用多条逻辑通道，避免死锁。
- **死锁避免**：通过限制路由路径（如维度序路由）来避免死锁。

## 9. 并行处理机与多处理机

### 9.1 并行处理机

- **SIMD 阵列处理机**：一个控制单元 + 多个处理单元（PE）。
  - 所有 PE 同时执行相同指令，处理不同数据。
  - 适合规则数据并行计算（矩阵运算、图像处理）。
- **典型结构**：ILLIAC IV、CM-2。

### 9.2 多处理机

**按耦合度分类**：
- **紧耦合（共享存储）**：多处理机共享同一主存，通信通过共享变量。即 SMP。
- **松耦合（分布存储）**：每台处理机有自己的存储器，通信通过消息传递。

**典型结构**：
- **SMP（对称多处理机）**：所有处理器平等，共享存储器和 I/O。
- **MPP（大规模并行处理机）**：成百上千个处理节点，分布式存储，高速互连网络。
- **集群（Cluster）**：用商品化网络连接独立计算机，性价比高。

### 9.3 多处理机的 Cache 一致性

**问题**：多个处理器各自的 Cache 中可能存有同一主存块的多个副本，一个处理器修改后其他处理器的副本失效。

**解决协议**：
- **监听协议（Snooping）**：通过总线监听其他处理器的操作。
  - **写无效（Write Invalidate）**：写时通知其他 Cache 使对应块无效。
  - **写更新（Write Update）**：写时将新值广播给其他 Cache。
- **基于目录的协议（Directory-based）**：用目录记录每个块的共享状态，适合大规模系统。
  - **MSI 协议**：Modified（已修改）、Shared（共享）、Invalid（无效）。
  - **MESI 协议**：增加 Exclusive（独占）状态。

### 9.4 多处理机性能模型

- **并行加速比** = T(1) / T(p)
- **并行效率** = 加速比 / p
- **Amdahl 定律的并行版本**：加速比 = 1 / ((1-f) + f/p)
- **Gustafson 定律**：解决问题规模随处理器数增大的情况，加速比 = f·p + (1-f)
- **等效率模型**：保持效率不变，问题规模需要随处理器数增长的函数。

### 9.5 多处理机操作系统

- **主从式**：一个主处理器管理所有 I/O 和调度。
- **浮动式**：任何处理器都可成为管理处理器。
- **独立式**：每个处理器运行自己的操作系统副本。
